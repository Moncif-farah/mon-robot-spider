01 — Composants Électroniques
🧠
ESP32 — Microcontrôleur Principal
ESP32-WROOM-32 / ESP32 DevKit v1
Cerveau du robot. Il exécute le code C++, gère la communication I2C avec le PCA9685 et dispose d'un module WiFi intégré permettant le contrôle sans fil via un navigateur web.
CPU240 MHz
WiFi802.11 b/g/n
Flash4 MB
GPIO36 pins
Tension3.3V / 5V
I2CSDA:21 SCL:22
// JUSTIFICATION : L'ESP32 est choisi pour son WiFi intégré (essentiel au contrôle distant), sa double cœur 240MHz, son support natif I2C et son excellent écosystème Arduino. Alternatives écartées : Arduino Nano (pas de WiFi), Raspberry Pi (Linux, trop consommant, boot lent).
⚡
PCA9685 — Contrôleur PWM 16 Canaux
Adafruit PCA9685 / Module I2C 12-bit
Génère les signaux PWM pour les 8 servomoteurs via I2C. Décharge les timers internes de l'ESP32 et offre une résolution 12-bit pour des mouvements précis.
Canaux16 PWM
Fréquence50 Hz
InterfaceI2C
Résolution12 bits
Addr I2C0x40
Logique5V VCC
// JUSTIFICATION : 8 servos requièrent 8 sorties PWM simultanées. L'ESP32 n'a que 4 timers hardware libres. Le PCA9685 déleste cette contrainte via seulement 2 fils I2C (SDA/SCL) et offre 16 canaux supplémentaires pour une future expansion.
🔩
Servomoteurs × 8 (SG90 / MG90S)
9g Micro Servo — 180° PWM Control
8 servos contrôlent les 4 pattes : 4 pour la rotation horizontale (axe Z, canaux 0-3) et 4 pour l'élévation verticale (axe X, canaux 4-7).
Couple1.8 kg/cm
Angle0–180°
Poids9 g
PWM Min150 (0°)
PWM Max600 (180°)
Tension4.8–6V
// MAPPING CANAUX : CH0→Base FL | CH1→Base FR | CH2→Base BL | CH3→Base BR | CH4→Jambe FL | CH5→Jambe FR | CH6→Jambe BL | CH7→Jambe BR. FL=Avant-Gauche, FR=Avant-Droit, BL=Arrière-Gauche, BR=Arrière-Droit.
🔋
Batterie LiPo 3S — 11.1V
LiPo 3S 1000–2200mAh / Connecteur XT30
Alimentation principale haute densité énergétique. Les 3 cellules en série délivrent 11.1V nominal, abaissé par le hacheur avant d'alimenter les servos et l'électronique.
Tension11.1V (3S)
Cellules3 × 3.7V
Capacité~1500mAh
Décharge20-30C
Poids~100g
Sortie→ Hacheur
// CALCUL COURANT : 8 servos SG90 × 600mA (charge) = 4.8A crête. LiPo 1500mAh 20C → 30A continu disponible. Autonomie estimée : ~20 min en marche active. Choisir un taux de décharge ≥ 15C.
⚙️
Hacheur — Régulateur Buck DC-DC
LM2596 / MP2307 Step-Down Converter
Abaisse la tension 11.1V de la batterie à 5–6V pour les servos et le PCA9685. Efficacité ~90% contre ~35% pour un régulateur linéaire.
Entrée11.1V
Sortie5–6V réglable
Courant3–5A
Efficacité~90%
TypeBuck DC-DC
ProtectionOVP / OCP
// JUSTIFICATION : Un régulateur linéaire 7805 dissiperait (11.1-5) × 5A = 30.5W en chaleur pure → impossible. Le buck convertit avec ~90% d'efficacité : chaleur minimale, tension servo stable même quand la batterie se décharge de 12.6V à 9V.
🔌
Interrupteur Principal ON/OFF
Rocker Switch SPST 10A / KCD1 Series
Coupe l'alimentation générale entre la batterie et le hacheur. Permet d'éteindre le robot sans déconnecter physiquement la batterie LiPo.
Courant max10A
Tension250V AC
TypeSPST ON/OFF
// PLACEMENT : Batterie (+) → Interrupteur → Hacheur (Vin). Choisir un switch ≥ 5A (les démarrages servo ont des pics importants). En mode OFF : consommation nulle, aucun courant de veille, protection contre décharge accidentelle.
02 — Conception Mécanique
🏗️
Châssis Principal
Plaque centrale rigide supportant toute l'électronique. Impression 3D en PLA ou PETG. Trous de montage M3 aux 4 coins pour les supports servo.
→ PLA : densité 1.24g/cm³, rigide, imprimable sans enceinte. Épaisseur recommandée : 4mm minimum. Inserts M3 thermoplastiques pour démontage.
🦾
Coxa (Segment 1)
Premier segment de patte monté sur le servo de base. Assure la rotation horizontale. Longueur : 20–30mm selon l'encombrement.
→ Trop court : couple réduit. Trop long : collision entre pattes adjacentes. Optimum géométrique calculé avec stepLength=30°.
🦿
Tibia (Segment 2)
Segment inférieur actionné par le servo de jambe. Contrôle l'élévation. Longueur 40–50mm pour une bonne portée de pas.
→ liftHeight=30° dans le code → levée de 30° au-dessus du home. Le tibia doit dégager le sol d'au moins 5mm lors de la levée.
🦶
Pied / Effecteur
Extrémité en contact sol. Embout TPU shore A 90 pour adhérence et amortissement des vibrations lors des impacts.
→ TPU élastique évite les glissements. Forme hémisphérique recommandée : contact ponctuel = moins de friction latérale.
⚓
Support Servo
Pièce en L reliant le châssis aux servomoteurs. Résiste aux couples de réaction lors des mouvements brusques.
→ Épaisseur min 3mm. Vis M2 autotaraudeuses dans les oreilles plastique du servo. 4 vis par servo pour rigidité optimale.
🔗
Cornière / Coupleur
Connecte deux segments de patte. Design en U pour distribuer les contraintes. Léger et rigide.
→ PLA renforcé fibres ou carbone 1mm. Éviter ABS (retrait = jeu mécanique). Joindre avec la collerette du servo palonier inclus.

Code & Software

Code source annoté ligne par ligne. Chaque bloc est expliqué dans le panneau de droite. Explorez aussi le contrôle WiFi et les algorithmes de marche.

spider_robot.ino — Arduino IDE (ESP32 Board)
#include <Wire.h>
#include <Adafruit_PWMServoDriver.h>
// 01 — Bibliothèques
#include <Wire.h> charge la bibliothèque I2C d'Arduino. Elle permet la communication entre l'ESP32 et le PCA9685 avec seulement 2 fils : SDA (données) et SCL (horloge).

#include <Adafruit_PWMServoDriver.h> charge la bibliothèque Adafruit pour le PCA9685. Elle abstrait tous les registres bas niveau et expose des méthodes simples comme setPWM() et setPWMFreq(). À installer depuis le Gestionnaire de Bibliothèques Arduino. 
Adafruit_PWMServoDriver pwm = Adafruit_PWMServoDriver();
// 02 — Objet Pilote PWM
Instancie un objet global pwm de type Adafruit_PWMServoDriver. Sans argument, il utilise l'adresse I2C par défaut 0x40 (les jumpers A0-A5 du PCA9685 doivent être ouverts). Cet objet est la référence principale pour toutes les commandes servo du programme. 
#define SERVO_MIN  150  // Impulsion min = 0°
#define SERVO_MAX  600  // Impulsion max = 180°
#define FREQUENCY   50  // Fréquence PWM servo (Hz)
// 03 — Constantes PWM
SERVO_MIN = 150 : valeur de tick (sur 4096) correspondant à 0° (~600µs d'impulsion).

SERVO_MAX = 600 : valeur correspondant à 180° (~2400µs). Ces valeurs sont spécifiques aux SG90/MG90S — ajuster légèrement si vos servos ne vont pas jusqu'aux butées.

FREQUENCY = 50 : 50Hz = période 20ms, standard universel pour les servos RC. Ne pas dépasser 60Hz pour les servos analogiques. 
// Canaux PCA9685 pour chaque patte
const int BASE_SERVOS[4] = {0, 1, 2, 3};
const int  LEG_SERVOS[4] = {4, 5, 6, 7};

// Index de patte : 0=FL, 1=FR, 2=BL, 3=BR
// 04 — Mapping des Canaux
Deux tableaux associent chaque patte à ses canaux PCA9685 :

BASE_SERVOS[] → canaux 0-3 : servent la rotation horizontale (avant/arrière de la patte dans le plan horizontal).

LEG_SERVOS[] → canaux 4-7 : servent l'élévation (levée/abaissement de la patte).

Exemple : BASE_SERVOS[0] = canal 0 = servo de base de la patte Avant-Gauche (FL). 
int stepDelay  = 150;  // ms entre mouvements
int liftHeight = 30;   // degrés de levée patte
int stepLength = 30;   // amplitude du pas (degrés)
int baseHome   = 90;   // position centrale servo base
int legHome    = 90;   // position centrale servo jambe
// 05 — Paramètres de Marche
Variables globales contrôlant la cinématique de marche :

stepDelay=150 : délai en ms. Augmenter pour mouvements plus lents/fluides. Diminuer pour vitesse maximale (risque de décrochage servo).

liftHeight=30 : hauteur de levée. La patte monte de 30° au-dessus du home (90°-30°=60°).

stepLength=30 : ±15° autour du centre pour chaque pas.

baseHome=legHome=90 : centre mécanique = position neutre debout. 
void setup() {
  Serial.begin(115200);
  Serial.println("Mini Spider Robot Init");

  Wire.begin(21, 22);      // SDA=21, SCL=22
  pwm.begin();
  pwm.setOscillatorFrequency(27000000);
  pwm.setPWMFreq(FREQUENCY); // 50 Hz

  standUp();
  delay(2000);
}
// 06 — Fonction setup()
Exécutée une seule fois au démarrage :

Serial.begin(115200) : ouvre le port série pour le débogage (moniteur série Arduino à 115200 bauds).

Wire.begin(21, 22) : initialise le bus I2C avec les GPIO hardware de l'ESP32. GPIO21=SDA, GPIO22=SCL.

pwm.begin() : réveille le PCA9685 depuis son mode sleep par défaut.

setOscillatorFrequency(27000000) : calibre l'oscillateur à 27MHz pour compenser les dérives de timing. Valeur empirique recommandée par Adafruit.

setPWMFreq(50) : configure 50Hz sur tous les 16 canaux.

standUp() suivi de delay(2000) : positionne le robot debout 2 secondes avant de démarrer la marche. 
void loop() {
  walkForward();
}
// 07 — Boucle Principale loop()
Appelée en boucle infinie après setup(). Actuellement, elle ne fait que marcher. C'est ici qu'on ajoutera la logique de contrôle : lecture des commandes WiFi, détection d'obstacles, sélection de mode. Exemple futur : if(command == "walk") walkForward(); else if(command == "stop") standUp(); 
void setServoAngle(int channel, int angle) {
  angle = constrain(angle, 0, 180);
  int pulse = map(angle, 0, 180,
                  SERVO_MIN, SERVO_MAX);
  pwm.setPWM(channel, 0, pulse);
}
// 08 — Fonction Centrale : setServoAngle()
Fonction de conversion angle → commande PWM :

constrain(angle, 0, 180) : clamp de sécurité. Protège mécaniquement le servo contre les angles hors plage (risque de casse de l'engrenage).

map(angle, 0, 180, 150, 600) : interpolation linéaire. 0°→150 ticks, 90°→375 ticks, 180°→600 ticks.

pwm.setPWM(ch, 0, pulse) : envoie la commande I2C. Premier arg=canal, second=délai montée (toujours 0), troisième=valeur de tick (0-4095). 
void standUp() {
  for (int i = 0; i < 4; i++) {
    setServoAngle(BASE_SERVOS[i], baseHome);
    setServoAngle( LEG_SERVOS[i], legHome);
  }
}
// 09 — Position Debout : standUp()
Boucle sur les 4 pattes (i = 0 à 3). Pour chaque patte, envoie les 2 servos (base et jambe) à leur position home de 90°.

Résultat : toutes les pattes sont perpendiculaires au corps, à mi-hauteur — position "debout" neutre et stable. C'est aussi la position de repos utilisée après la commande STOP. 
void walkForward() {
  // Creep gait : 1 patte à la fois
  moveLeg(0); // Front Left
  moveLeg(3); // Back Right
  moveLeg(1); // Front Right
  moveLeg(2); // Back Left

  // Propulsion : tous les servos base simultanés
  for (int i = 0; i <= stepLength; i += 5) {
    // Pattes gauches (0,2) : reculent (↓ angle)
    setServoAngle(BASE_SERVOS[0],
      (baseHome + stepLength/2) - i);
    setServoAngle(BASE_SERVOS[2],
      (baseHome + stepLength/2) - i);
    // Pattes droites (1,3) : reculent (↑ angle)
    setServoAngle(BASE_SERVOS[1],
      (baseHome - stepLength/2) + i);
    setServoAngle(BASE_SERVOS[3],
      (baseHome - stepLength/2) + i);
    delay(20);
  }
}
// 10 — Marche Avant : walkForward()
Implémente le Creep Gait (marche rampante) :

Phase 1 — Repositionnement : chaque patte se déplace dans les airs, une par une, dans l'ordre diagonal FL→BR→FR→BL. Les 3 pattes restantes restent au sol = stabilité statique garantie à tout moment.

Phase 2 — Propulsion : une fois les 4 pattes repositionnées, la boucle for fait glisser tous les servos de base simultanément, propulsant le corps vers l'avant. Les pattes gauches et droites tournent en sens opposés car leurs servos sont montés en miroir.

delay(20) : 20ms par incrément de 5° = mouvement fluide sans à-coups. 
void moveLeg(int legIndex) {
  // Direction "avant" selon côté gauche/droit
  int forwardAngle = (legIndex==0||legIndex==2)
    ? (baseHome + stepLength/2)
    : (baseHome - stepLength/2);

  // 1. LEVER la patte
  setServoAngle(LEG_SERVOS[legIndex],
    legHome - liftHeight); // 90-30 = 60°
  delay(stepDelay / 2);

  // 2. AVANCER dans les airs
  setServoAngle(BASE_SERVOS[legIndex],
    forwardAngle);
  delay(stepDelay / 2);

  // 3. POSER sur le sol
  setServoAngle(LEG_SERVOS[legIndex],
    legHome); // retour à 90°
  delay(stepDelay);
}
// 11 — Un Pas Complet : moveLeg()
Exécute les 3 phases d'un pas pour la patte legIndex :

1. LEVER — legHome - liftHeight = 60°. Le servo de jambe lève la patte. delay(stepDelay/2) = 75ms laisse le servo atteindre sa position.

2. AVANCER — le servo de base pivote vers la position forwardAngle. La direction dépend du côté : gauche (0,2) va vers 90+15=105°, droite (1,3) vers 90-15=75° (servos montés en miroir).

3. POSER — legHome = 90° : la jambe descend pour toucher le sol. delay(stepDelay) = 150ms : temps complet pour amortir l'atterrissage.
WiFi — Architecture de Contrôle

Extension du firmware avec un serveur HTTP intégré à l'ESP32. Le robot crée son propre réseau WiFi (Access Point) et expose une interface REST + page HTML de contrôle.

📡
Mode Access Point
L'ESP32 crée son propre réseau "SpiderRobot". Connectez votre téléphone ou PC à ce réseau WiFi, puis ouvrez 192.168.4.1 dans un navigateur pour accéder à l'interface.
🌐
Mode Station (STA)
Connectez l'ESP32 à votre routeur WiFi existant en renseignant SSID et mot de passe dans le code. L'IP locale est attribuée par votre box — consultez le moniteur série pour la trouver.
Endpoints REST disponibles
GET/Interface HTML de contrôle
GET/walkDémarrer la marche avant
GET/stopArrêter → standUp()
GET/standPosition debout uniquement
POST/speedModifier stepDelay — body JSON: {"delay": 100}
POST/servoCommande servo directe — {"ch":0,"angle":90}
GET/liftleg?leg=0Lever une patte spécifique (0-3)
GET/turn?dir=leftRotation sur place (left/right)
spider_wifi.ino — Extension WebServer ESP32
#include <WiFi.h>
#include <WebServer.h>

const char* ssid     = "SpiderRobot";
const char* password = "spider123";

WebServer server(80);
bool isWalking = false;

void setupWiFi() {
  WiFi.softAP(ssid, password);
  Serial.print("AP IP: ");
  Serial.println(WiFi.softAPIP()); // 192.168.4.1

  server.on("/", HTTP_GET, handleRoot);
  server.on("/walk", HTTP_GET, []() {
    isWalking = true;
    server.send(200, "text/plain", "Walking");
  });
  server.on("/stop", HTTP_GET, []() {
    isWalking = false;
    standUp();
    server.send(200, "text/plain", "Stopped");
  });
  server.begin();
}

void loop() {
  server.handleClient(); // OBLIGATOIRE
  if (isWalking) walkForward();
}
// WiFi Access Point + Serveur HTTP
WiFi.softAP(ssid, password) : crée un point d'accès WiFi. L'ESP32 devient lui-même un routeur. IP fixe : 192.168.4.1.

server.on(route, method, callback) : associe une URL à une fonction lambda. Chaque route est un endpoint REST.

isWalking : flag booléen non-bloquant. Les requêtes HTTP modifient ce flag ; la boucle décide d'agir. Évite le blocage du serveur HTTP pendant la marche.

server.handleClient() : DOIT être appelé à chaque iteration de loop(). Sans cet appel, le serveur ne répond à aucune requête.
🦟
Creep Gait — Actuel
Une seule patte se déplace à la fois. Les 3 autres restent au sol = stabilité statique maximale. Idéal pour démarrer et tester le hardware.
  • Séquence : FL(0) → BR(3) → FR(1) → BL(2)
  • Stabilité statique à tout moment garantie
  • Vitesse ~10-20 cm/min avec stepDelay=150
  • Implémenté dans walkForward()
🕷️
Trot Gait — Avancé
Deux pattes diagonales se lèvent simultanément (FL+BR, puis FR+BL). Deux fois plus rapide mais nécessite une meilleure synchronisation des servos.
  • Paires diagonales : FL+BR | FR+BL
  • Stabilité dynamique (robot en mouvement)
  • Vitesse estimée ~35-40 cm/min
  • Nécessite tuning du stepDelay et timing
🎯
Lever une Patte (Tâche)
Lever et maintenir une patte spécifique en l'air. Utile pour démonstrations, franchissement d'obstacle ou manipulation d'objet léger.
  • Code : setServoAngle(LEG_SERVOS[n], 60)
  • Ne pas appeler standUp() après
  • Angle levée : legHome - liftHeight = 60°
  • Via WiFi : GET /liftleg?leg=0
🔄
Rotation Sur Place
Tourner le robot sans se déplacer. Toutes les pattes gauches poussent en avant, les droites reculent (ou inversement pour tourner dans l'autre sens).
  • Pattes G (0,2) : baseHome + stepLength
  • Pattes D (1,3) : baseHome - stepLength
  • Inverser pour tourner à droite
  • Via WiFi : GET /turn?dir=left ou right
