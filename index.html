Composant Référence Rôle & Justification
ESP32 ESP32-WROOM-32 Microcontrôleur principal. WiFi intégré pour le contrôle sans fil, 240 MHz dual-core, support I2C natif sur GPIO 21/22. Choisi pour son WiFi intégré vs Arduino (pas de WiFi) et son écosystème Arduino complet.
PCA9685 Adafruit 16-Ch PWM Driver PWM 16 canaux via I2C. Génère les signaux 50Hz pour les 8 servos en soulageant les timers internes de l'ESP32. Résolution 12 bits. Adresse I2C : 0x40.
Servomoteurs × 8 SG90 / MG90S 4 servos de base (canaux 0–3, rotation horizontale) + 4 servos de jambe (canaux 4–7, élévation). Couple 1.8 kg/cm, PWM 150–600, tension 5–6V, poids 9g chacun.
Batterie LiPo 3S 11.1V / ~1500 mAh Source d'alimentation principale. Haute densité énergétique, 3 cellules 3.7V en série. Décharge 20C minimum pour alimenter les pics de courant servo (estimation 4.8A crête sous charge).
Hacheur Buck LM2596 / MP2307 Convertit 11.1V → 5–6V pour les servos. Efficacité ~90% contre ~35% pour un régulateur linéaire. Un 7805 dissiperait 30W de chaleur — inadmissible. Sortie réglable, protection OVP/OCP.
Interrupteur SPST Rocker 10A Coupe l'alimentation entre la batterie et le hacheur. Sécurité : évite la décharge accidentelle. Courant nominal ≥ 5A pour absorber les pics au démarrage.
Châssis + pattes PLA / PETG — FDM Corps, coxa, tibia et effecteurs imprimés en 3D. PLA : rigidité élevée, densité 1.24 g/cm³. Effecteurs TPU Shore A90 pour adhérence sol. Épaisseur châssis ≥ 4 mm.
spider_robot.ino C++ / Arduino 
#include <Wire.h>
#include <Adafruit_PWMServoDriver.h>

// Objet pilote PCA9685 (adresse I2C 0x40 par défaut)
Adafruit_PWMServoDriver pwm = Adafruit_PWMServoDriver();

// Constantes PWM — ajuster selon vos servos
#define SERVO_MIN  150   // 0 degrés
#define SERVO_MAX  600   // 180 degrés
#define FREQUENCY   50   // Hz — standard servo
// Imports & configuration
Wire.h : bibliothèque I2C Arduino. Permet la communication avec le PCA9685 sur 2 fils (SDA/SCL).

SERVO_MIN/MAX : plage de ticks (sur 4096 total) correspondant à 0° et 180°. Valeurs typiques SG90 : 150–600. Ajuster si le servo ne va pas en butée.
Mapping canaux & paramètres 
// BASE_SERVOS : rotation horizontale (axe Z)
// LEG_SERVOS  : élévation verticale (axe X)
// Index : 0=FL, 1=FR, 2=BL, 3=BR
const int BASE_SERVOS[4] = {0, 1, 2, 3};
const int  LEG_SERVOS[4] = {4, 5, 6, 7};

int stepDelay  = 150;  // ms entre mouvements
int liftHeight = 30;   // degrés de levée
int stepLength = 30;   // amplitude d'un pas
int baseHome   = 90;   // position neutre
int legHome    = 90;
// Mapping & paramètres cinématiques
Deux tableaux associent chaque patte (FL, FR, BL, BR) à ses deux canaux PCA9685 : un pour la rotation horizontale (base) et un pour l'élévation (jambe).

stepDelay=150ms : délai entre chaque mouvement. Augmenter pour plus de fluidité, réduire pour plus de vitesse.

liftHeight=30° : la patte monte de 90°→60° lors de la levée.
setup() & setServoAngle() 
void setup() {
  Serial.begin(115200);
  Wire.begin(21, 22);       // SDA=21, SCL=22
  pwm.begin();
  pwm.setOscillatorFrequency(27000000);
  pwm.setPWMFreq(FREQUENCY);
  standUp();
  delay(2000);
}

void setServoAngle(int ch, int angle) {
  angle = constrain(angle, 0, 180);
  int pulse = map(angle, 0, 180, SERVO_MIN, SERVO_MAX);
  pwm.setPWM(ch, 0, pulse);
}
// Initialisation & commande servo
Wire.begin(21, 22) : démarre le bus I2C sur les GPIO hardware ESP32. GPIO21=SDA, GPIO22=SCL.

setOscillatorFrequency(27000000) : calibre l'oscillateur PCA9685 à 27 MHz pour compenser les dérives de timing internes.

setServoAngle : convertit un angle 0–180° en tick PWM via interpolation linéaire, avec protection hardware (constrain).
walkForward() & moveLeg() 
// Creep gait : 1 patte à la fois → stabilité statique
void walkForward() {
  moveLeg(0); moveLeg(3);
  moveLeg(1); moveLeg(2);

  for (int i = 0; i <= stepLength; i += 5) {
    setServoAngle(BASE_SERVOS[0], (baseHome+stepLength/2)-i);
    setServoAngle(BASE_SERVOS[2], (baseHome+stepLength/2)-i);
    setServoAngle(BASE_SERVOS[1], (baseHome-stepLength/2)+i);
    setServoAngle(BASE_SERVOS[3], (baseHome-stepLength/2)+i);
    delay(20);
  }
}

void moveLeg(int idx) {
  int fwd = (idx==0||idx==2)
    ? baseHome+stepLength/2
    : baseHome-stepLength/2;
  // 1. Lever
  setServoAngle(LEG_SERVOS[idx], legHome-liftHeight);
  delay(stepDelay/2);
  // 2. Avancer dans les airs
  setServoAngle(BASE_SERVOS[idx], fwd);
  delay(stepDelay/2);
  // 3. Poser
  setServoAngle(LEG_SERVOS[idx], legHome);
  delay(stepDelay);
}
// Algorithme de marche
Creep Gait : une seule patte se lève à la fois, les 3 autres restent au sol. Stabilité statique garantie à tout moment. Séquence diagonale : FL→BR→FR→BL.

Phase propulsion : après repositionnement des 4 pattes, tous les servos de base se déplacent simultanément en sens opposé selon le côté (montage miroir gauche/droite).

moveLeg : 3 phases — lever (60°), avancer en l'air (forwardAngle), poser (90°).
