Mini Spider Robot

Documentation technique du robot araignée miniature (Quadrupède). Ce projet englobe la conception CAO, l'assemblage électronique de précision et le développement d'un firmware orienté objet C++ avec un serveur API WiFi intégré sur un ESP32.

Architecture Matérielle

ESP32 DevKit v1
Microcontrôleur WiFi
Cerveau du robot. Gère la logique cinématique, la communication I2C, et héberge le point d'accès WiFi (Serveur Web Asynchrone) pour le contrôle à distance. Choisi pour son double cœur et son horloge à 240 MHz.
PCA9685
Driver PWM 16 Canaux
Indispensable pour générer simultanément et précisément 8 signaux PWM à 50Hz pour les servomoteurs (posture & marche), libérant ainsi les timers matériels de l'ESP32. Connecté en I2C.
Unité de Puissance
LiPo 3S 11.1V + Buck regulateur
Un convertisseur Step-Down hacheur abaisse la tension de 11.1V à 5.5V pour fournir un courant critique (jusqu'à 5 ampères crête) aux servos lors des démarrages de la marche, avec un excellent rendement énergétique.

Conception du Firmware

Le code écrit en C++ orchestre la cinématique inverse (IK) simplifiée et la gestion des requêtes asynchrones en environnement critique de temps (temps réel souple).

spider_cinematics.cpp (Extrait du déplacement)
#include <Wire.h>
#include <Adafruit_PWMServoDriver.h>

Adafruit_PWMServoDriver pwm = Adafruit_PWMServoDriver(0x40);

// Constantes de calibrage de marche
const int HOME_BASE = 90;
const int HOME_LEG = 90;
const int LIFT_ANGLE = 30;

// Exécution séquentielle de mouvement segmentaire
void stepLeg(int legId, int targetHeading) {
  // 1. Phae de levée (Dégagement du sol)
  setServoAngle(legId, SERVO_LEG, HOME_LEG - LIFT_ANGLE);
  delay(100); // Attente de la stabilisation mécanique
  
  // 2. Phase de rotation de la base (Orientation Z)
  setServoAngle(legId, SERVO_BASE, targetHeading);
  delay(100);
  
  // 3. Phase d'atterrissage
  setServoAngle(legId, SERVO_LEG, HOME_LEG);
}
